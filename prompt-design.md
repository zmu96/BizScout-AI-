📄 프롬프트 설계 문서

BizScout – AI 기업 리서치 요약 웹 서비스

1. 프롬프트 설계 목적

본 프로젝트에서는 여러 웹사이트에서 수집한 **비정형 데이터(HTML, 텍스트, JSON)**를
AI 모델이 **웹 서비스에 바로 활용 가능한 구조화된 데이터(JSON)**로 변환하는 것을 목표로 하였다.

이를 위해 다음과 같은 요구사항을 충족하는 프롬프트를 설계하였다.

불필요한 설명 없이 JSON만 출력

프론트엔드에서 바로 렌더링 가능한 고정된 데이터 구조

다양한 입력 형식에도 일관된 출력 포맷 유지

2. 사용한 프롬프트 예시

아래는 실제 프로젝트에서 사용한 핵심 프롬프트 설계 내용이다.

너는 웹 서비스용 데이터 생성기다.

아래는 여러 웹사이트에서 수집한 원본 데이터이다.
(JSON, HTML, 텍스트가 섞여 있을 수 있다)

이 데이터를 분석해서
반드시 아래 JSON 형식으로만 출력하라.
설명, 문장, 마크다운, 텍스트는 절대 출력하지 마라.

출력 JSON 형식:

{
  "basicInfo": {
    "companyName": "",
    "industry": "",
    "headquarters": "",
    "founded": ""
  },
  "sentiment": {
    "positive": [],
    "negative": []
  },
  "newsSummary": "",
  "insight": [],
  "jobs": [
    {
      "title": "",
      "link": ""
    }
  ]
}

원본 데이터:
{{ $json.trimmed_data }}

3. 프롬프트 설계 의도 분석
3.1 웹 서비스 연계 고려

출력 결과는 React 기반 웹 페이지에서 바로 사용 가능

프론트엔드 컴포넌트 구조와 1:1로 매칭되는 JSON 설계

API 응답 형태를 미리 가정한 프롬프트 구조

3.2 불완전한 데이터 대응

일부 정보가 존재하지 않더라도
빈 문자열 또는 빈 배열로 구조 유지

다양한 사이트 조합에도 UI 깨짐 방지

3.3 로컬 LLM 환경 최적화

과도한 추론을 요구하지 않는 단순·명확한 지시

토큰 사용량 최소화를 위한 불필요한 출력 차단

로컬 LLM(Ko-LLM 계열)의 안정적인 JSON 출력 유도

4. 프롬프트 설계의 장점

✅ 일관된 출력 포맷 유지

✅ 프론트엔드 연동 용이

✅ 서비스 확장 시 API 구조 변경 최소화

✅ 모델 교체(OpenAI / 로컬 LLM) 시 재사용 가능

5. 프롬프트의 단점
kullm-r-i1 모델을 로컬 환경에서 사용했으나 위의 프롬프트로도 온전히 json 형태의 답만 나오는 경우는 드물었음.
따라서 모델에서 거쳐 나온 출력을 한번 더 조정해 자바스크립트 코드로 <think> 부분을 제거하고 json 형태의 출력만 남도록 조정이 필요했다.  

6. 정리

본 프로젝트의 프롬프트 설계는
단순 텍스트 요약이 아닌 웹 서비스 운영을 전제로 한 데이터 생성을 목표로 하였으며,
실제 서비스 환경에서 활용 가능한 구조를 중심으로 설계되었다.
